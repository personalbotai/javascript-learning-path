Pengenalan dan Sejarah JavaScript

Apa Itu JavaScript?

JavaScript adalah bahasa pemrograman high-level, interpreted, dan multi-paradigm yang awalnya dibuat untuk menambahkan interaktivitas ke halaman web. Saat ini, JavaScript digunakan di:

  • Frontend: Browser (DOM manipulation, events)
  • Backend: Node.js (server-side applications)
  • Mobile: React Native, Ionic
  • Desktop: Electron, Tauri
  • IoT: Johnny-Five, Espruino

Sejarah Singkat

1995 - Lahirnya JavaScript

Diciptakan oleh Brendan Eich di Netscape dalam waktu 10 hari. Nama awalnya Mocha, lalu LiveScript, dan akhirnya JavaScript untuk memanfaatkan popularitas Java.

1997 - ECMAScript Standard

JavaScript distandardkan menjadi ECMAScript (ES) oleh Ecma International. Versi pertama adalah ES1.

2009 - ES5

ES5 menambahkan fitur seperti JSON, Array.prototype.forEach, dan strict mode. Ini adalah fondasi modern JavaScript.

2015 - ES6/ES2015

Perubahan terbesar dalam sejarah JS: let/const, arrow functions, classes, promises, modules, template literals, dan banyak lagi.

2016 - Present - Yearly Releases

ECMAScript sekarang rilis tahunan (ES2016, ES2017, ... ES2024). Fitur-fitur baru ditambahkan secara bertahap seperti async/await, optional chaining, nullish coalescing, dan top-level await.

JavaScript vs ECMAScript

ECMAScript adalah spesifikasi bahasa (standard), sedangkan JavaScript adalah implementasi dari spesifikasi tersebut. Browser dan Node.js mengimplementasikan ECMAScript dengan tambahan API spesifik lingkungan mereka.

ECMAScript (Standard)

  • Sintaks bahasa
  • Tipe data
  • Control flow
  • Functions & objects
  • Prototype inheritance

JavaScript (Implementation)

  • ECMAScript core
  • DOM API (browser only)
  • BOM (Browser Object Model)
  • Node.js APIs (fs, http, etc)
  • Built-in objects (console, setTimeout)

Ciri-Ciri JavaScript

Dynamically Typed

Tipe data ditentukan saat runtime, bukan saat compile time.

Prototype-Based OOP

Warisan dilakukan melalui prototype chain, bukan kelas (sebelum ES6).

First-Class Functions

Functions adalah nilai pertama-class: bisa disimpan di variabel, dikembalikan, dan dilewatkan sebagai argumen.

Closure Support

Function bisa "mengingat" environment di mana ia dibuat.

Event-Driven & Non-blocking I/O

Menggunakan event loop untuk concurrency tanpa threading (terutama di Node.js).

Tantangan JavaScript

  • Type Coercion: Perilaku konversi tipe otomatis yang bisa membingungkan ("5" + 3 === "53", "5" - 3 === 2)
  • this keyword: Binding dinamis yang bergantung pada cara function dipanggil
  • Hoisting: Variabel dan function dinaikkan ke atas scope
  • Equality: Perbedaan antara == dan === (strict equality direkomendasikan)
  • Callback Hell: nested callbacks yang sulit dibaca (diselesaikan dengan promises & async/await)

Try It Yourself

Coba kode berikut di console browser (F12) atau Node.js REPL:

// Tampilkan informasi JavaScript
console.log("Hello, JavaScript!");
console.log("Version:", navigator.userAgent); // di browser
console.log("Year:", new Date().getFullYear());

// Contoh type coercion
console.log("5" + 3); // "53"
console.log("5" - 3); // 2
console.log(5 == "5"); // true
console.log(5 === "5"); // false

// Contoh closure
function makeCounter() {
    let count = 0;
    return function() {
        return ++count;
    };
}
const counter = makeCounter();
console.log(counter()); // 1
console.log(counter()); // 2
